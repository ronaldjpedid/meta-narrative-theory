# Peer Review Guide

## 1. Purpose & Scope
Peer review ensures clarity, logic, testability, and ethical integrity across all contributions to the meta-narrative theory.

## 2. Review Criteria
- Clarity and accessibility
- Logical consistency and rigor
- Testability and falsifiability
- Ethical alignment (restoration, agency, abolition of harm)
- Adherence to coding and documentation standards

## 3. How to Leave Feedback
- Use inline comments on pull requests
- Open issues for larger concerns
- Suggest improvements or alternatives

## 4. Approval & Change Requests
- Approve if all criteria are met
- Request changes with clear, actionable feedback
- Collaborate respectfully to resolve issues

## 5. Common Pitfalls
- Unclear or inaccessible language
- Unreferenced claims or missing objections
- Magic numbers/strings in code
- Incomplete documentation or missing tests

## 6. Proposing Improvements
- Suggest new protocols, guides, or models via pull request
- Reference relevant sections or objections
- Encourage open discussion and iteration
