# Onboarding Guide

Welcome to the Meta-Narrative Theory Project!

## 1. Project Overview & Mission
This project formalizes a universal meta-narrative system—protocols, equations, laws, arguments, guides, and abolishments—for reality, story, and agency. Our mission is to create a testable, adaptable, and ethical framework for all contributors.

## 2. Folder Structure & Key Files
- **arguments/**: Core arguments and objections
- **math/**: Mathematical models and equations
- **laws/**: Universal laws and principles
- **protocols/**: Actionable protocols and abolishments
- **guides/**: Guides for all walks of life
- **hypotheses/**: Testable hypotheses
- **predictions/**: Predictive models
- **reflection/**: Project reflections and lessons
- **code/**: TypeScript models and logic

## 3. How to Contribute
- Fork the repo and create a feature branch
- Add or update markdown, TypeScript, or guides
- Follow coding and documentation standards
- Submit a pull request for review

## 4. Coding & Documentation Standards
- Use TypeScript (.ts) for all code
- Use markdown (.md) for documentation
- No magic numbers/strings—use constants
- All user-facing strings should be localizable
- Follow DRY and SOP principles

## 5. Using aegis-flash & Deno CLI
- Use aegis-flash for CLI, compression, and encryption tasks
- All tests and automation run through Deno CLI

## 6. Submitting Changes
- Use clear commit messages
- Reference related issues or guides
- Ensure all tests pass before submitting

## 7. Where to Find Help
- See the README and guides in this folder
- Open an issue or discussion for questions

- [ ] Read the README and onboarding guide
- [ ] Set up Deno and aegis-flash
- [ ] Explore the folder structure
